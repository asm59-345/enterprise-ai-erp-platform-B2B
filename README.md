# Stratos ERP — Clean Minimalist Monolith Suite

Welcome to **Stratos ERP**, a highly polished, production-grade corporate administration system powered by an AI-augmented Monolith. Stratos unifies key ERP pipelines such as HRMS, General Ledger Finance, CRM coordinate pipelines, global multi-region Inventory systems, and real-time observability telemetry into a single-page reactive console.

---

## 🎨 Design Philosophy & Aesthetic
- **Times New Roman Typography**: Built using high-contrast, elegant serif typography across all screens, tables, and telemetry terminals to ensure a distinctive, formal, and authoritative presentation.
- **Adaptive Theme Engine**: Reads client operating system preferences dynamically on boot. Users can manually lock overrides (`Light Mode`, `Dark Mode`, or `System Default`) via the header bar or login view.
- **Full Responsiveness**: Seamlessly handles any device screen size (from compact mobile views to ultra-wide desktop monitors). Incorporates an adaptive sidebar menu that collapses into an animated side drawer on mobile screens.

---

## 🛠️ Feature Breakdown

### 1. Unified Authentication Gate
- Supports both **manual mock developer sandbox** mode and **live Google Firebase Authentication** flows.
- Clean minimalist entry forms featuring inline client validation, loading state, and error handling.

### 2. Live Telemetry & Observability Hub
- Real-time CPU core saturation, node memory allocation, transaction latency tracker, and connection pool dashboards.
- **Interactive Scalability Stress Simulator**: Adjust simulated client thread counts to initiate load testing. Watch the built-in system logger spawn Kubernetes Pod replicas, archive telemetry files, and load-balance incoming request volumes live.

### 3. Interactive System Q&A Hub
- Category-filtered FAQ database explaining system mechanics, billing deactivations, multi-agent AI debates, and sharding.
- **Support Assistant Input Form**: Ask custom questions about the codebase or roles, and receive precise responses generated by the dynamic search lookup engine.

### 4. About & Walkthrough Section
- Architectural visual blueprint highlighting Express router, Vite asset servers, and Firestore.
- Built-in video mockup showing tutorial walkthrough instructions.

## ⚡ Deployment & Integration Configuration

To link your live **Cloud Firestore database** or **Firebase Authentication** nodes, define the standard environment variables inside your `.env` configuration file:

```env
# Firebase configuration secrets
VITE_FIREBASE_API_KEY="your-api-key-here"
VITE_FIREBASE_AUTH_DOMAIN="your-auth-domain-here"
VITE_FIREBASE_PROJECT_ID="your-project-id"
VITE_FIREBASE_STORAGE_BUCKET="your-storage-bucket"
VITE_FIREBASE_MESSAGING_SENDER_ID="sender-id"
VITE_FIREBASE_APP_ID="app-id"
```

*Note: If no Firebase configuration keys are supplied, Stratos automatically activates local sandboxed memory fallback adaptors, ensuring continuous, seamless functionality and client-side local storage replication.*
